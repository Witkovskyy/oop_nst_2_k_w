A feature-\/rich Chess \doxylink{class_engine}{Engine} and Graphical User Interface built with {\bfseries{C++17}} and {\bfseries{SFML}}. This project demonstrates advanced Object-\/\+Oriented Programming concepts, custom AI algorithms, and multithreaded architecture.\hypertarget{md__r_e_a_d_m_e_autotoc_md1}{}\doxysection{\texorpdfstring{Features}{Features}}\label{md__r_e_a_d_m_e_autotoc_md1}
\hypertarget{md__r_e_a_d_m_e_autotoc_md2}{}\doxysubsection{\texorpdfstring{Core Game Mechanics}{Core Game Mechanics}}\label{md__r_e_a_d_m_e_autotoc_md2}

\begin{DoxyItemize}
\item Full implementation of chess rules (movement, captures, check, checkmate, stalemate).
\item {\bfseries{\doxylink{struct_move}{Move} Validation:\+}} Robust system to prevent illegal moves (e.\+g., moving into check).
\end{DoxyItemize}\hypertarget{md__r_e_a_d_m_e_autotoc_md3}{}\doxysubsection{\texorpdfstring{AI Engine}{AI Engine}}\label{md__r_e_a_d_m_e_autotoc_md3}
The project features a custom-\/built chess engine capable of playing against a human:\+
\begin{DoxyItemize}
\item {\bfseries{Negamax Algorithm:\+}} Simplified minimax variant for move evaluation.
\item {\bfseries{Alpha-\/\+Beta Pruning:\+}} Optimizes the search tree by eliminating irrelevant branches.
\item {\bfseries{Quiescence Search:\+}} Extends the search at leaf nodes to avoid the "{}horizon effect"{} during captures.
\item {\bfseries{Zobrist Hashing:\+}} Efficient board state hashing for fast lookups.
\item {\bfseries{Transposition Table (TT):\+}} Caches search results to avoid re-\/evaluating the same positions (64MB default size).
\item {\bfseries{Evaluation Function:\+}} Uses Material balance and Piece-\/\+Square Tables (PST) for positional scoring.
\item {\bfseries{Static Exchange Evaluation (SEE):\+}} heuristic to determine if a capture is profitable.
\end{DoxyItemize}\hypertarget{md__r_e_a_d_m_e_autotoc_md4}{}\doxysubsection{\texorpdfstring{Architecture \& Design}{Architecture \& Design}}\label{md__r_e_a_d_m_e_autotoc_md4}

\begin{DoxyItemize}
\item {\bfseries{Object-\/\+Oriented Design:\+}} Polymorphic {\ttfamily \doxylink{class_piece}{Piece}} hierarchy ({\ttfamily \doxylink{class_pawn}{Pawn}}, {\ttfamily \doxylink{class_rook}{Rook}}, {\ttfamily \doxylink{class_knight}{Knight}}, etc.).
\item {\bfseries{Design Patterns:\+}}
\begin{DoxyItemize}
\item {\bfseries{Singleton:\+}} Used for the {\ttfamily \doxylink{class_async_logger}{Async\+Logger}} to ensure a single logging point.
\item {\bfseries{RAII:\+}} Resource management for SFML assets and memory.
\end{DoxyItemize}
\item {\bfseries{Multithreading:\+}}
\begin{DoxyItemize}
\item {\bfseries{Async Logger:\+}} A dedicated thread processes log messages from a queue to prevent I/\+O blocking in the main game loop.
\item Thread-\/safe operations using {\ttfamily std::\+mutex} and {\ttfamily std::\+condition\+\_\+variable}.
\end{DoxyItemize}
\end{DoxyItemize}\hypertarget{md__r_e_a_d_m_e_autotoc_md5}{}\doxysubsection{\texorpdfstring{Tech Stack}{Tech Stack}}\label{md__r_e_a_d_m_e_autotoc_md5}

\item {\bfseries{Language:\+}} C++17
\item {\bfseries{Graphics:\+}} SFML (Simple and Fast Multimedia Library) 2.\+6+
\item {\bfseries{Build System:\+}} CMake
\item {\bfseries{Testing:\+}} Catch2 (Unit Tests \& Coverage)
\end{DoxyItemize}
